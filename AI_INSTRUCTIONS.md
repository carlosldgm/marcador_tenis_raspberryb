# AI_INSTRUCTIONS.md

Cuando modifiques este proyecto:

- No cambies nombres de archivos sin pedir confirmación.
- No rompas la compatibilidad con el protocolo ESP-NOW.
- No modifiques la API Flask existente salvo que sea necesario.
- Mantén la lógica del tenis dentro de logic.py.
- Prioriza soluciones simples y mantenibles.
- Comenta el código sólo cuando aporte valor.
- Antes de realizar cambios grandes, explica el impacto.
- Evita duplicar código.
- Sigue el estilo ya existente en el proyecto.
- Si detectas una mejora arquitectónica, propónla antes de implementarla.
